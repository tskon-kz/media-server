# Media Server

> [English version](README.md)

Jellyfin + qBittorrent + Telegram-бот на одном сервере.

## Установка

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/tskon-kz/media-server/main/install.sh)
```

Скрипт начинает с выбора языка интерфейса, затем устанавливает Docker если его нет, скачивает нужные файлы в `~/media-server/`, запрашивает настройки, автоматически конфигурирует qBittorrent и Jellyfin, запускает контейнеры.

### Что спрашивает установщик

| Вопрос | Описание |
|--------|----------|
| Язык | English или Русский |
| Telegram Bot Token | От [@BotFather](https://t.me/BotFather) |
| Твой Telegram ID | От [@userinfobot](https://t.me/userinfobot). Через запятую для нескольких пользователей |
| IP сервера | Внешний IP для ссылок в боте |
| Путь к media | Хостовый путь, монтируемый как `/media` (дефолт: `./media`). Укажи путь к диску, например `/mnt/disk2` |
| Логин / пароль Jellyfin | Создаётся при первом запуске |
| Название сервера Jellyfin | Необязательно, по умолчанию `Media Server` |
| Прокси для Telegram | Необязательно, например `socks5://user:pass@host:port` |
| Кастомные порты | Необязательно — нажми `n` чтобы использовать дефолты (Jellyfin: 8096, qBittorrent: 8080) |

### Ручная установка

Если не хочешь использовать установщик:

```bash
git clone https://github.com/tskon-kz/media-server ~/media-server
cd ~/media-server
cp /dev/null .env
```

Добавь в `.env`:

```dotenv
BOT_TOKEN=...          # от @BotFather
ALLOWED_USER=123456789 # от @userinfobot; через запятую для нескольких пользователей
WATCHTOWER_TOKEN=...   # любая случайная строка, например: openssl rand -hex 16

# Необязательно — раскомментируй чтобы переопределить дефолты
# JELLYFIN_PORT=8096
# QB_PORT=8080
# MEDIA_PATH=./media
```

Затем заполни БД значениями, которые установщик собирает интерактивно:

```bash
DB=bot-data/media_server.db
mkdir -p bot-data
python3 -c "
import sqlite3, sys
db = sqlite3.connect('$DB')
db.execute('CREATE TABLE IF NOT EXISTS config (key TEXT PRIMARY KEY, value TEXT)')
db.executemany('INSERT OR REPLACE INTO config VALUES (?,?)', [
    ('server_ip',  'IP_СЕРВЕРА'),
    ('lang',       'ru'),
    ('proxy_url',  ''),           # необязательно: socks5://user:pass@host:port
])
db.commit()
"
docker compose up -d
```

После запуска контейнеров настрой qBittorrent и Jellyfin через их WebUI, затем сохрани credentials:

```bash
python3 -c "
import sqlite3
db = sqlite3.connect('$DB')
db.executemany('INSERT OR REPLACE INTO config VALUES (?,?)', [
    ('qb_user',          'admin'),
    ('qb_pass',          'ПАРОЛЬ_QB_WEBUI'),  # Tools → Options → Web UI
    ('jellyfin_api_key', 'JELLYFIN_API_KEY'),
])
db.commit()
"
```

Jellyfin API key: Dashboard → API Keys → +.

## Работа с ботом

### Добавление торрентов

Отправь в чат **magnet-ссылку** или **.torrent-файл** — бот спросит в какую категорию сохранить. Когда скачивание завершится, придёт уведомление и Jellyfin автоматически просканирует библиотеку.

### Команды

| Команда | Действие |
|---------|----------|
| `/list` | Список торрентов со статусами и кнопками |
| `/status` | Текущие скорости скачивания / раздачи |
| `/scan` | Запустить сканирование библиотек Jellyfin |
| `/settings` | Меню настроек |

### `/list`

- Иконки статусов: ⬇️ скачивает, ✅ готово, ⏸ на паузе, 🌱 сидирует, ❌ ошибка
- **Режим редактирования** — кнопки на каждый торрент: 🗑 удалить с файлами, 📁 переместить в категорию, 🔄 перепарсить файлы

### `/settings`

| Раздел | Что можно сделать |
|--------|-------------------|
| Категории | Добавить, переименовать, удалить. Каждая категория — папка и библиотека в Jellyfin |
| Язык | Переключить между русским и английским |
| Пользователи Jellyfin | Добавить и удалить аккаунты Jellyfin |
| Обновление | Проверить и применить обновление в один тап |
| Быстрые ссылки | Открыть qBittorrent / Jellyfin прямо из меню |

### Автопереименование медиафайлов

Когда торрент завершает скачивание, бот создаёт **хардлинки** по путям, которые Jellyfin понимает как стандартные, — без изменения исходных файлов (qBittorrent продолжает раздавать как ни в чём не бывало).

- **Сериалы** → `Название/Season 01/Название - S01E04.mkv`
- **Фильмы** → `Название (2023)/Название (2023).mkv`

Имена файлов разбирает [guessit](https://guessit.readthedocs.io). Если файл не удаётся распознать автоматически, бот присылает сообщение с вариантами:
- **Ввести вручную** — напиши `S01E04` (или `Название (Год)` для фильмов), хардлинк будет создан
- **Оставить как есть** — создать плоский хардлинк без переименования
- **Пропустить** — не трогать файл, хардлинк не создавать

При удалении торрента через бот все его хардлинки удаляются автоматически.

### Обновления

Бот проверяет новую версию при запуске и каждые 6 часов, присылает уведомление. Чтобы обновить вручную: `/settings` → **Обновление** → **Обновить**. Бот перезапустится автоматически примерно через 1 минуту.

### Категории

По умолчанию: **Movies** (`/media/movies`) и **Series** (`/media/series`).

При добавлении категории создаётся библиотека в Jellyfin, при удалении — удаляется. Доступные типы: Фильмы, Сериалы, Музыка, Смешанная.

## CI/CD

При пуше в `main`, затрагивающем `bot/`, `lang/` или `pyproject.toml`, запускаются два воркфлоу:

**`build-push.yml`** — собирает Docker-образ и пушит в `ghcr.io` с тегами `:latest` и `:<version>`. Никаких дополнительных секретов не нужно, хватает дефолтного `GITHUB_TOKEN`.

**`dev-deploy.yml`** — запускается после успешного билда, подключается по SSH и вызывает Watchtower HTTP API для обновления сервера. Нужен только мейнтейнеру проекта для своего сервера — в форках можно игнорировать или удалить. Требует секреты:

| Секрет | Значение |
|--------|----------|
| `SERVER_HOST` | IP сервера |
| `SERVER_USER` | Пользователь SSH |
| `SERVER_SSH_KEY` | Приватный SSH ключ |
| `SSH_PORT` | SSH порт |

Оба воркфлоу можно запустить вручную через **Actions → Run workflow**.

## Перемещение медиатеки

```bash
bash migrate-media.sh
```

Останавливает контейнеры, копирует всё через rsync с отображением прогресса, обновляет `MEDIA_PATH` в `.env`, перезапускает контейнеры. Перед стартом проверяет свободное место.

## Сброс

```bash
bash teardown.sh
```

Останавливает контейнеры (опционально удаляет Docker-образы), удаляет `data/`, `bot-data/`, `.env`. Опционально удаляет `media/`.

## Доступ

- Jellyfin: `http://IP_СЕРВЕРА:8096`
- qBittorrent: `http://IP_СЕРВЕРА:8080`
