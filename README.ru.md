# Media Server

> [English version](README.md)

Jellyfin + qBittorrent + Telegram-бот на одном сервере.

## Быстрый старт

```bash
git clone https://github.com/tskon-kz/mediaserver.git ~/media-server
cd ~/media-server
bash setup.sh
```

`setup.sh` устанавливает Docker если его нет, запрашивает все настройки, автоматически конфигурирует qBittorrent и Jellyfin, запускает контейнеры. Можно запустить повторно для перенастройки.

### Что спрашивает setup

| Вопрос | Описание |
|--------|----------|
| Telegram Bot Token | От [@BotFather](https://t.me/BotFather) |
| Твой Telegram ID | От [@userinfobot](https://t.me/userinfobot). Через запятую для нескольких пользователей |
| IP сервера | Внешний IP для ссылок в боте |
| Пароль qBittorrent | Пароль WebUI (логин: `admin`) |
| Логин / пароль Jellyfin | Создаётся автоматически при первом запуске |
| Название сервера Jellyfin | Необязательно, по умолчанию `Media Server` |
| Прокси для Telegram | Необязательно, например `socks5://user:pass@host:port` |
| Путь к media | Хостовый путь, монтируемый как `/media` (дефолт: `./media`). Укажи путь к диску, например `/mnt/disk2` |
| Кастомные порты | Необязательно — нажми `n` чтобы пропустить всё и использовать дефолты (Jellyfin: 8096, qBittorrent: 8080) |

### Переменные .env

```dotenv
# Обязательные
BOT_TOKEN=...
ALLOWED_USER=123456789          # Telegram ID через запятую
SERVER_IP=1.2.3.4

# Необязательные
PROXY_URL=socks5://user:pass@host:port
MEDIA_PATH=./media              # хостовый путь, монтируется как /media во всех контейнерах
MEDIA_BASE=/media               # базовый путь внутри контейнера для подсказок бота
JELLYFIN_PORT=8096
QB_PORT=8080
```

`JELLYFIN_API_KEY` добавляется автоматически во время setup — вручную не задавать.

## Работа с ботом

### Добавление торрентов

Отправь в чат **magnet-ссылку** или **.torrent-файл** — бот спросит в какую категорию сохранить. Когда скачивание завершится, придёт уведомление и Jellyfin автоматически просканирует библиотеку.

### Команды

| Команда | Действие |
|---------|----------|
| `/list` | Список торрентов со статусами и кнопками |
| `/status` | Текущие скорости скачивания / раздачи |
| `/scan` | Запустить сканирование библиотек Jellyfin |
| `/settings` | Меню настроек |

### `/list` подробнее

- Иконки статусов: ⬇️ скачивает, ✅ готово, ⏸ на паузе, 🌱 сидирует, ❌ ошибка и другие
- **Режим редактирования** — кнопки на каждый торрент: 🗑 удалить (вместе с файлами) и 📁 переместить в категорию

### Меню `/settings`

| Раздел | Что можно сделать |
|--------|-------------------|
| Категории | Добавить, переименовать, удалить. Каждая категория — папка и библиотека в Jellyfin |
| Язык | Переключить между русским и английским |
| Пароль qBittorrent | Сменить пароль WebUI |
| Пользователи Jellyfin | Добавить и удалить аккаунты Jellyfin |
| Быстрые ссылки | Открыть qBittorrent / Jellyfin прямо из меню |

### Категории

По умолчанию: **Movies** (`/media/movies`) и **Series** (`/media/series`).

При добавлении категории в Jellyfin создаётся соответствующая библиотека. При удалении — удаляется. Доступные типы библиотек: Фильмы, Сериалы, Музыка, Смешанная.

## CI/CD

Добавь секреты в репозиторий: **Settings → Secrets and variables → Actions**

| Секрет | Значение |
|--------|----------|
| `SERVER_HOST` | IP сервера |
| `SERVER_USER` | Пользователь SSH |
| `SERVER_SSH_KEY` | Приватный SSH ключ |
| `SSH_PORT` | SSH порт |

Каждый пуш в `main` подтягивает код на сервере и перезапускает контейнер бота.

## Сброс

```bash
bash teardown.sh
```

Останавливает контейнеры (опционально удаляет Docker-образы), удаляет `data/`, `.env` и конфиги бота (`creds.json`, `lang.json`, `categories.json`). Опционально удаляет `media/`. Файлы репозитория не трогает.

## Доступ

- Jellyfin: `http://IP_СЕРВЕРА:8096` (или кастомный `JELLYFIN_PORT`)
- qBittorrent: `http://IP_СЕРВЕРА:8080` (или кастомный `QB_PORT`)
